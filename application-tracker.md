# Application Tracker

Current as of 2026-06-01.

Purpose: track a small, high-quality market test. Do not use this as a spray-and-pray list.

## Weekly Target

- 5 quality applications.
- 3 warm/recruiter messages.
- 1 feedback action: mentor, peer review, mock screen, or recruiter call.
- 1 update to CV/LinkedIn/scripts based on market feedback.

## Status Values

Use these exact statuses:

- `Saved`
- `Applied`
- `Warm outreach sent`
- `Recruiter replied`
- `Screen booked`
- `Interviewing`
- `Rejected`
- `No response`
- `Paused`
- `Closed / expired`

## Fit Ratings

- `A`: strong .NET/SaaS/integrations/workflow fit, realistic timezone/location.
- `B`: decent technical fit, some mismatch in stack, seniority, or region.
- `C`: useful calibration only; do not apply unless there is a specific reason.

## Application Quality Checklist

Before applying:

- Re-open the job link and confirm it is still live.
- Confirm location, timezone, work authorization, and remote policy.
- Identify the top 5 repeated requirements from the ad.
- Adjust the CV summary or first bullets only if the role is `A` or strong `B`.
- Save the exact role URL and a copy/PDF/screenshot if the role matters.
- Note whether the role is `.NET backend`, `.NET full-stack`, `Vue/Nuxt`, `integrations/platform`, or `workflow automation`.

After applying:

- Set follow-up date to 5-7 days later.
- Save the submitted CV version.
- Record any recruiter/hiring-manager names.
- If rejected or ignored, write one reason guess only. Do not overinterpret one data point.

## Weekly Review

Answer these every week:

- Which role family got the most positive signal?
- Which requirement repeated most often that is missing or weak?
- Which geography/timezone looked most realistic?
- Which CV/LinkedIn wording needs changing?
- Which roles should be downgraded from target to calibration-only?

## Open Placeholders

- Public location wording.
- Legal work authorization by country/region.
- Preferred salary/rate bands.
- Exact CV version naming once applications begin.
- Public links: LinkedIn, GitHub, portfolio, RefineBridge, sanitized case studies.
