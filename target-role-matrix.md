# Target Role Matrix v1

Current as of 2026-06-01.

Purpose: calibrate target roles from current real job ads. This is not a mass-application list. Role ads can expire quickly; manually re-check each link before applying.

Positioning used:

> Backend-leaning .NET/product engineer with visible Vue/Nuxt full-stack capability, strongest around SaaS, integrations, workflow automation, background jobs, and remote-first product delivery.

## Research Notes

- Sources used: company career pages and live job-board pages found through web search on 2026-06-01.
- Geographic spread preserved: UK/Europe overlap, Australia/New Zealand-friendly overlap, North America-only calibration roles, South Africa/EMEA, Canada/CET, and global-ish remote examples.
- This matrix deliberately includes some weak fits so the pattern is visible: region lock, too senior, non-Vue frontend, heavy cloud/IoT, or hybrid/right-to-work constraints.
- Strongest current signal: practical .NET SaaS/product engineering with full-stack contribution, APIs, SQL, cloud, testing, remote collaboration, and product ownership.

## Repeated Requirements Summary

Most repeated technical requirements:

- C#/.NET/.NET Core or ASP.NET Core.
- REST APIs; sometimes GraphQL.
- SQL experience: SQL Server, PostgreSQL, or mixed SQL/NoSQL.
- Entity Framework or ORM familiarity.
- Modern frontend capability: React, Angular, Vue, Nuxt, or TypeScript.
- Cloud familiarity, especially Azure for .NET roles; AWS appears often in product/SaaS roles.
- Testing: unit, integration, TDD language, code review, maintainability.
- CI/CD, Git, deployment awareness.
- Production debugging, performance, reliability, and security awareness.
- SaaS/product context, customer impact, cross-functional collaboration, and ownership.
- Remote communication: written English, proactive updates, async/distributed collaboration.

Most repeated seniority signals:

- 3+ years for mid-level/full-stack roles.
- 5-6+ years for senior product/full-stack roles.
- 7-8+ years where roles expect mentoring, architecture, cloud depth, or technical leadership.
- Senior roles often ask for mentoring, architectural decision-making, code review, testing standards, and production reliability.

Most repeated domain patterns:

- SaaS platforms.
- Workflow automation.
- API integrations and external platform constraints.
- Internal/admin tooling.
- Cloud-hosted products.
- Document/data/business-process platforms.
- Background or event-driven processing.

## Grouped Requirements

### Must-have for the first market test

- A clear `.NET/C#` anchor in CV, LinkedIn, and outreach.
- Comfortable explanation of ASP.NET Core, REST APIs, EF/SQL, background jobs, and integration failure handling.
- Visible full-stack capability through Vue/Nuxt/TypeScript, while staying backend-leaning.
- One strong RDrive DataBridge case study focused on external integrations, configurable workflows, background processing, and operational visibility.
- One RefineBridge case study focused on modern .NET/Nuxt/Postgres SaaS patterns, auth/account boundaries, billing/webhook awareness, and product/admin workflows.
- Remote collaboration story: how to communicate status, tradeoffs, blockers, and decisions while working independently.
- Testing/reliability story: unit/integration tests, logs, retries, health checks, support visibility, or production debugging.
- Practical SQL and data-modeling confidence.

### Nice-to-have to develop next

- React comfort, because many .NET full-stack roles pair .NET with React instead of Vue.
- Azure confidence: App Service, Functions, Service Bus/Event Grid, Key Vault, Azure SQL, Azure DevOps.
- Docker/container basics; Kubernetes only at concept level initially.
- Messaging/event-driven patterns: queues, retries, idempotency, duplicate handling.
- DDD/modular architecture vocabulary used cautiously and tied to real examples.
- TDD/integration testing language, especially when senior roles ask for quality ownership.
- GraphQL/MongoDB familiarity as recognition-level knowledge, not a headline skill.
- Security interview basics: auth vs authorization, tenant/account boundaries, secrets, webhook trust, PII/logging.

### Ignore-for-now / do not over-optimize yet

- FAANG-style algorithm-heavy preparation unless target ads start requiring it.
- Dynamic programming, graph-heavy LeetCode, and large-scale distributed systems theory as the main prep path.
- Pure frontend specialization. Vue/Nuxt should be visible, but not the whole identity.
- Deep DevOps/SRE/Kubernetes ownership. Useful vocabulary, not primary positioning.
- Staff/principal roles and roles requiring org-wide architecture leadership.
- Power Platform certifications unless that path becomes a deliberate target.
- Exact frontend framework matching. React is common, but the stronger market story is `.NET product engineer who can ship full-stack`.
- Industry-specific expertise in insurance, payroll, construction, logistics, or IoT. Treat these as context ramps, not blockers.

## Ranked Role-Family Fit

1. Full-stack .NET product engineer in SaaS/platform companies.
   - Best evidence fit: Gearset, saas.group, CoverGo, ShiftForce, AIA Contract Documents.
   - Why: combines .NET backend, product delivery, APIs, admin workflows, cloud, testing, and practical frontend work.

2. Backend-leaning .NET integrations/platform engineer.
   - Best evidence fit: Flatgigs/Ahoy, Mesh Systems, SPOTIO as calibration, Gearset as product-platform overlap.
   - Why: matches RDrive DataBridge strongly: external systems, background jobs, API boundaries, reliability, status visibility.

3. .NET + Vue/Nuxt full-stack engineer.
   - Best evidence fit: Deel, CoverGo, Celara, Kingscrowd as Vue/Nuxt calibration.
   - Why: validates keeping Vue/Nuxt visible. However, the market often says React/Angular, so do not narrow to Vue-only.

4. Workflow automation / internal tools / Microsoft ecosystem engineer.
   - Best evidence fit: European Dynamics, ShiftForce, AIA Contract Documents.
   - Why: fits the "turn messy business workflows into configurable software" story.

5. Remote mid-level full-stack/product engineer.
   - Best evidence fit: Kingscrowd, Pyyne, ShiftForce if location-compatible.
   - Why: potentially safer than jumping straight into senior roles while interview readiness is still untested.

6. Senior cloud/backend specialist.
   - Best evidence fit: Mesh Systems, SPOTIO, Vista Group, J.D. Power.
   - Why: useful to understand senior market expectations, but should not be the initial default because they may over-index on cloud scale, domain-specific data, or formal senior-team experience.

## Calibration Decisions

- Keep the public headline close to: `Backend-Leaning .NET Engineer | SaaS Workflows, Integrations, Background Jobs`.
- In applications, vary the first line by role:
  - `.NET full-stack/product engineer` for SaaS/product ads.
  - `.NET backend/integrations engineer` for API/platform/background-job ads.
  - `.NET/Vue/Nuxt engineer` only when Vue/Nuxt is in the ad.
- Do not present as pure senior by default. Use experienced ownership language, then apply selectively to senior roles where the ad values autonomy/product delivery more than formal people leadership.
- React should be treated as an adaptation target, not a blocker. Many good .NET SaaS roles use React.
- UK/Europe remote remains strongest strategically if right-to-work/timezone works. Australia/NZ-friendly roles exist but often have local work-rights or hybrid constraints.
- US/Canada roles are useful for calibration but many are not directly actionable from Thailand.

## Manual Checks Before Applying

- Re-open every source link and confirm the ad is still live.
- Confirm work authorization and timezone feasibility before investing in tailored applications.
- Check whether remote means "anywhere", "country only", "region only", or "hybrid after probation".
- Save a PDF/screenshot of any role worth applying to because job ads disappear quickly.
- For each selected role, tailor CV bullets toward the ad's repeated words: `.NET`, `SaaS`, `APIs`, `integration`, `background jobs`, `testing`, `cloud`, `remote collaboration`, `Vue/Nuxt` or `React`.
